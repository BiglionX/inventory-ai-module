📋 模块简介

产品库是行业标准产品主数据中心，作为公开标准数据源（行业统一），为整个系统提供标准化的产品基础数据支持。采用五库设计（品牌库、整机库、配件库、部件库、零件库），支持BOM管理和全生命周期溯源追踪。

📢 模块定位声明

核心理念：产品库是"行业标准产品主数据中心"，作为公开标准数据源（行业统一），为整个系统提供标准化的产品基础数据支持。

简单来说：产品库就像是一个"产品字典"，里面记录了所有产品的标准信息。

💡 打个比方：就像手机通讯录，产品库存储的是"联系人基本信息"（姓名、电话），而进销存系统记录的是"你和这个联系人的交易记录"（买了什么、什么时候买的）。

✨ 核心功能

🏪 品牌管理

管理产品品牌信息，包括Logo、描述、网站、联系方式等，支持API密钥配置

💻 整机产品

管理完整的成品产品信息和规格参数，支持SKU编码、状态管理、版本控制

🔧 配件管理

管理产品配件信息和兼容性，支持兼容产品列表、规格参数

⚙️ 部件管理

管理产品组成部件（如CPU、内存、硬盘），支持类型分类、技术规格

🔩 零件管理

管理最小物理单元的零件（如螺丝、电阻、电容），支持材质、尺寸信息

📱 溯源码

生成和管理产品溯源码，追踪产品从生产到售后的全生命周期

🔗 BOM管理

物料清单管理，定义产品之间的关联关系（包含、兼容、组成等）

📥 数据导入

从CSV、Excel或API批量导入产品数据，支持任务记录和错误日志

🏗️ 架构设计

产品库采用清晰的三层架构设计：

1️⃣ 产品库

作用：存储产品的标准信息

特点：全局共享，所有人用同一份数据

例子：iPhone 15的规格、参数、图片

2️⃣ 进销存系统

作用：管理你的库存和交易

特点：每个用户有自己的独立数据

例子：你仓库里有10台iPhone 15

3️⃣ 溯源码插件

作用：追踪每个产品的生命周期

特点：给每个产品一个唯一身份证

例子：这台iPhone 15的生产、销售、维修记录

⚠️ 重要提示：产品库采用单一全局实例设计，通过品牌、类目等维度组织数据，不需要也不支持创建多个数据库实例。这样设计的好处是保证数据的一致性和标准化。

💼 典型使用场景

场景1：电子产品零售商

  • 在产品库中查找"Apple"品牌的所有iPhone型号
  • 查看iPhone 15的详细规格和配件清单
  • 将产品信息导入到你的进销存系统
  • 为你的每台iPhone生成溯源码，方便售后追踪

场景2：电脑组装商

  • 在部件库中查找各种CPU、内存、硬盘
  • 查看哪些配件兼容特定的主板
  • 使用BOM功能管理整机的组成部件
  • 为组装好的电脑生成完整的产品档案

场景3：维修服务商

  • 扫描产品溯源码，快速查看产品信息
  • 查询零件库，找到替换零件的详细信息
  • 记录维修历史，完善产品生命周期档案
  • 为客户提供透明的维修记录和零件来源

❓ 常见问题（FAQ）

Q1: 为什么不能创建多个产品库？

A: 产品库的定位是"行业标准数据"，就像字典一样，大家都应该使用同一本标准字典。如果你需要隔离数据，应该在进销存系统中实现，那里支持多租户隔离。

Q2: 产品库和进销存有什么区别？

A: 产品库存储的是"产品是什么"（名称、规格、图片等标准信息），进销存记录的是"我有多少"（库存数量、采购价格、销售记录等业务数据）。

Q3: 我需要自己录入所有产品信息吗？

A: 不一定！你可以：

  • 使用数据导入功能批量导入CSV/Excel文件
  • 通过API从品牌方官方数据源同步
  • 手动添加少量特殊产品

Q4: 什么是BOM？有什么用？

A: BOM（Bill of Materials）是物料清单，用来描述一个产品由哪些部件组成。比如一台电脑包含：1个CPU、2根内存条、1个硬盘等。BOM可以帮助你了解产品的完整组成结构、计算生产成本、管理配件兼容性。

Q5: 溯源码怎么使用？

A: 溯源码就像是产品的"身份证"，每个产品都有唯一的二维码。使用时：

  1. 在产品库中选择要追踪的产品
  2. 生成溯源码（可以批量生成）
  3. 打印二维码贴到产品上
  4. 客户扫码即可查看产品完整信息和使用历史

📦 快速开始

⚙️ 环境要求

  • Node.js 18+ - JavaScript运行时
  • PostgreSQL 12+ - 或使用Supabase云数据库
  • Supabase CLI - 数据库迁移工具

1. 执行数据库迁移

# 创建产品库Schema
supabase db push --file supabase/migrations/020_create_product_library_schema.sql

# 配置访问权限
supabase db push --file supabase/migrations/027_grant_product_library_permissions.sql

2. 配置环境变量

确保 .env.local 中包含：

NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key

3. 启动开发服务器

npm run dev
# 访问 http://localhost:3001/product-library

4. 开始使用

  • 首先创建品牌信息
  • 然后添加整机产品和配件/部件/零件
  • 配置BOM关系（可选）
  • 发布产品到产品库
  • 生成溯源码并追踪产品生命周期
